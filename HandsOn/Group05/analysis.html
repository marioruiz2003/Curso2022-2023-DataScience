Resource Naming Strategy

There’s two kinds of URIs:
- URIs to define the ontology.
- URIs to define initialized resources.

URIs to define the ontology:

The URIs for the ontology constitute the model that we are going to use, and they use the following prefixes:

The “base” prefix is used to build most ontologies, which follow the following model: “http://findeducationalcenter.linkeddata.es/lcc/ontology/EducationalCenter#[ontologies]”, where “ontologies” is replaced by any class or property, for example " http://findeducationalcenter.linkeddata.es/lcc/ontology/EducationalCenter#EducationalCenter ", which is a class.

The “schema” prefix will also be used for various already existing properties, such as “email” (schema:email) or the telephone (schema:telephone). Also, we have used other already existing properties or class names from other sources like wgs84 (wgs84:SpatialThing)

The URIs to define the instances.

The instances are the resources that we have, and in the creation of these we will use the following prefix:

@prefix resource: http://findeducationalcenter.linkeddata.es/lcc/resource/ .
And the schema of your usage will be prefix + ClassName + InstanceName

Where ClassName is the name of the class and InstanceName is an instance. An example would be: http://findeducationalcenter.linkeddata.es/lcc/resource/EducationalCenter/1, an instance with id equals 1, of the EducationalCenter class.
